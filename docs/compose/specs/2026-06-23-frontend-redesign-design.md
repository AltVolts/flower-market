# Редизайн фронтенда магазина цветов

## [S1] Проблема

Текущий фронтенд выглядит как шаблонный макет:
- Нет визуальной иерархии
- Базовые стили без современных паттернов
- Нет анимаций и переходов
- Простые формы без визуальной обратной связи
- Используются placeholder изображения

## [S2] Целевой дизайн

**Стиль:** Тёплый уютный с пастельными тонами

**Цветовая палитра:**
- Основной: `#E8B4B8` (мягкий розовый)
- Акцент: `#9CB99C` (мягкий зелёный)
- Фон: `#FDF8F5` (тёплый белый)
- Текст: `#5D4E37` (тёплый коричневый)
- Вторичный: `#F5E6D3` (кремовый)

**Шрифты:**
- Заголовки: Montserrat (жирный)
- Тело: Open Sans (обычный)

## [S3] Структура страниц

### Главная страница
1. **Герой-блок** —.full-width баннер с текстом и CTA
2. **Популярные товары** — горизонтальная прокрутка или сетка 4 колонки
3. **Категории** — карточки с иконками и названиями
4. **О нас** — текст с фоном
5. **Контакты** — краткая информация

### Каталог
- Сайдбар с фильтрами по категориям
- Сетка товаров 3 колонки (десктоп), 2 (планшет), 1 (мобайл)
- Карточки товаров с hover-эффектом

### Страница товара
- Галерея изображений (левая часть)
- Информация о товаре (правая часть)
- Кнопка "Заказать"

### Форма заказа
- Простая форма с валидацией
- Визуальная обратная связи при ошибках

## [S4] Компоненты

### Header
- Фиксированный верхний бар
- Логотип слева, навигация справа
- Мобильное меню (бургер)

### Footer
- Три колонки: О нас, Контакты, Режим работы
- Копирайт внизу

### ProductCard
- Изображение с hover-эффектом (увеличение)
- Название, цена, краткое описание
- Кнопка "Подробнее"

### CategoryFilter
- Горизонтальные табы или вертикальный список
- Визуальное выделение выбранной категории

## [S5] Анимации

- Hover-эффекты на карточках товаров (тень, увеличение)
- Плавные переходы между страницами
- Анимация появления элементов при скролле
- Визуальная обратная связи на кнопках

## [S6] Адаптивность

- Десктоп: полная ширина, 3-4 колонки
- Планшет: 2 колонки, адаптивный сайдбар
- Мобильный: 1 колонка, бургер-меню

## [S7] Требования

- Использовать существующий Tailwind конфиг
- Сохранить текущую структуру маршрутов
- Работать с существующим API
- Не добавлять новые зависимости (кроме шрифтов)
