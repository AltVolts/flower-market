# [S1] Problem

Микробизнес по выращиванию цветов в теплицах и сбору ягод/грибов в лесу продаёт продукцию на рынках и по предварительным заказам. Текущие проблемы:

- Клиенты теряются (нет базы клиентов)
- Заказы путаются (ручное управление)
- Нет привлечения новых клиентов (только устно на рынке)
- Нет системы управления каталогом

Масштаб: 5-10 заказов в неделю. Техническая поддержка — владелец бизнеса.

# [S2] Solution overview

Создать сайт-витрину с каталогом товаров и формой заказа. Сайт размещается бесплатно (Vercel + Render), управляется через встроенную админ-панель.

**Ключевые особенности:**
- Связь с Марий Эл (природные цвета, виды края, история бизнеса)
- Управляемые категории товаров (добавление/удаление/сортировка)
- Простая форма заказа с уведомлением владельца

# [S3] Target audience

Жители Марий Эл и окрестностей, которые:
- Покупают цветы на рынке или заказывают по телефону
- Ищут свежие ягоды и грибы
- Хотят красивые кашпо и аксессуары

# [S4] Product categories

| Категория | Описание |
|-----------|----------|
| Цветы | Свежие цветы разного вида из теплиц |
| Кашпо и горшки | Цветы в горшках и кашпо, готовые композиции |
| Ягоды | Клубника, земляника, другие ягоды из леса/поля |
| Грибы | Собранные в лесу грибы |
| Аксессуары | Удобрения, инструменты, другие товары |

Категории управления из админ-панели: добавление, удаление, переименование, сортировка.

# [S5] Architecture

```
Пользователь (браузер)
       ↓
   React + Tailwind (фронтенд)
       ↓
   FastAPI (бэкенд)
       ↓
   SQLite (база данных)
```

**Деплой:**
- Фронтенд: Vercel (бесплатный tier)
- Бэкенд: Render (бесплатный tier)
- Домен: пока бесплатный URL, потом можно подключить свой

# [S6] Pages

## Главная страница
- Шапка: лого, название, контакты
- Герой-баннер: фото природы Марий Эл + цветы
- Блок "О нас": история бизнеса, привязка к Марий Эл
- Популярные товары (по 2-3 из каждой категории)
- Ссылки на категории

## Каталог
- Фильтр по категориям (боковое меню или табы)
- Сетка товаров: фото, название, цена
- Пагинация при большом количестве товаров

## Страница товара
- Крупное фото
- Название, цена, описание
- Наличие (в наличии / под заказ)
- Кнопка "Заказать"

## Форма заказа
- Список выбранных товаров
- Поля: имя, телефон, комментарий
- Кнопка "Отправить заказ"
- Сообщение об успешной отправке

## Контакты
- Телефон, адрес, время работы
- Карта проезда (Яндекс.Карты)
- Соцсети (если есть)

# [S7] Admin panel

Отдельный вход по паролю (без регистрации).

## Управление товарами
- Список товаров с фильтрацией по категориям
- Добавление нового товара:
  - Фото (загрузка с компьютера)
  - Название
  - Цена
  - Описание
  - Категория (выбор из списка)
  - Наличие (в наличии / под заказ)
- Редактирование существующего товара
- Удаление товара

## Управление категориями
- Список категорий в порядке отображения
- Добавление новой категории
- Переименование категории
- Удаление категории (с предупреждением о привязанных товарах)
- Сортировка (перетаскивание)

## Просмотр заказов
- Список заказов с фильтрацией по статусу и дате
- Информация о заказе: имя, телефон, товары, сумма, комментарий
- Изменение статуса: Новый → В работе → Выполнен

## Настройки
- Название магазина
- Контакты (телефон, адрес, время работы)
- Описание "О нас"
- Герой-баннер (загрузка фото)

# [S8] Order workflow

```
1. Клиент заходит на сайт
2. Смотрит каталог, выбирает товары
3. Нажимает "Заказать"
4. Заполняет форму: имя, телефон, комментарий
5. Заявка сохраняется в БД
6. Владелец получает уведомление (email)
7. Владелец связывается с клиентом для подтверждения
8. Клиент забирает заказ (самовывоз)
9. Владелец меняет статус на "Выполнен"
```

# [S10] Tech stack

| Компонент | Технология | Причина |
|-----------|------------|---------|
| Фронтенд | React 18 + Tailwind CSS | Современно, красиво, быстро |
| Бэкенд | FastAPI (Python) | Простой API, быстрый |
| БД | SQLite | Без сервера, хватит на долгое время |
| ORM | SQLAlchemy | Простой доступ к БД |
| Деплой (фронт) | Vercel | Бесплатный tier, автоматический деплой |
| Деплой (бэк) | Render | Бесплатный tier, поддержка Python |

# [S11] MVP scope

Для первой версии (MVP):

1. Главная страница с каталогом
2. Страница товара
3. Форма заказа
4. Админ-панель:
   - Управление товарами (CRUD)
   - Управление категориями (CRUD + сортировка)
   - Просмотр заказов
   - Настройки сайта

**Не входит в MVP:**
- Онлайн-оплата
- Доставка
- Рассылки
- Аналитика
- Мультиязычность

# [S12] Future enhancements

Когда MVP будет работать и появятся заказы:

- SQLite → PostgreSQL (если заказов станет 50+ в день)
- Добавить онлайн-оплату (ЮKassa, Тинькофф)
- Добавить доставку (интеграция с курьерами)
- Добавить Telegram-бота для уведомлений
- Добавить систему лояльности (скидки за повторные заказы)
- SEO-оптимизация
- Аналитика (что смотрят, что заказывают)
